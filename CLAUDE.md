# Roadly UI — Claude Code Context

## Project Purpose

Roadly is a web-based coding education platform. This repo is the React/Vite frontend. Users follow AI-generated learning roadmaps, write code in a browser IDE with live Docker containers, and get AI mentoring.

## Tech Stack

| Category | Technology |
|----------|-----------|
| Framework | React 19, Vite 7, TypeScript 5.9 |
| Styling | TailwindCSS 4, shadcn/ui (Radix), Lucide icons |
| State | Zustand 5 (complex state), Redux Toolkit 2 + RTK Query (server state) |
| Routing | React Router 7 |
| Editor | `@monaco-editor/react` 4.7 |
| Terminal | xterm.js 5 + WebSocket (socket.io-client) |
| Auth | AWS Amplify 6 / Cognito |
| Real-time | `@microsoft/signalr` 10 (SignalR hub) |
| Forms | react-hook-form + zod |

## Key Architectural Decisions

### State Management
- **Zustand** for UI/domain state within features (`useIdeStore`, `useChatStore`)
- **RTK Query** for all backend HTTP calls — define endpoints in `apiSlice.injectEndpoints()`
- Do **not** mix: no `useState` for state that belongs in Zustand; no Zustand for server cache

### API Layer
- All HTTP calls go through `src/store/api/apiSlice.ts` base query
- Base query injects Cognito JWT from `tokenProvider`
- Pattern: `apiSlice.injectEndpoints({ endpoints: builder => ({ myEndpoint: builder.query({...}) }) })`

### Routing
- Routes defined in `BasicDataRouter.tsx` using React Router 7
- Protected routes use auth guards from `features/authorization/`

### IDE Architecture
- Monaco editor models managed by `useMonacoModels` hook
- Terminal connects via WebSocket to `docker-pty-proxy`
- AI review sends code from Monaco to .NET API → Python agent

## Conventions

- **Components**: `PascalCase.tsx` in `features/<name>/components/`
- **Hooks**: `camelCase.ts` prefixed `use` in `features/<name>/hooks/`
- **Stores**: `useXxxStore.ts` (Zustand) in `features/<name>/store/`
- **API endpoints**: in `features/<name>/api/` or `store/api/`
- **Types**: in `features/<name>/types/`
- No `any` — use proper TypeScript types
- shadcn/ui components in `src/components/ui/` — do not modify them directly

## What NOT to Touch

- `src/components/ui/` — auto-generated shadcn/ui, re-run `shadcn add` to update
- `amplify_outputs.json` — auto-generated by AWS Amplify CLI
- `src/lib/amplify-config.ts` — only change if Cognito pool config changes
- Direct imports from `react-redux` — use `useReduxHooks.ts` typed wrappers instead

## Communication with Backend

| Purpose | Destination | Transport |
|---------|-------------|-----------|
| All CRUD | .NET API (port 5000) | RTK Query (REST) |
| AI Chat streaming | Python service (port 8000) | SSE POST via `@microsoft/fetch-event-source` |
| Terminal | Go Proxy (port 8080) | WebSocket |
| Real-time notifications | .NET SignalR hub | `@microsoft/signalr` |
